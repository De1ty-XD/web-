# 实验名称：2FA SINPLE BYPASS

## 基本信息
- **难度** 初级
- **类别** 身份验证

## 漏洞原理
- 服务器在验证2FA之前就已经建立了用户登录会话（session）
- 
## 攻击步骤
- 首先登录自己的账户 -> 进入账户页面观察记录当前URL `academy.net/my-account?id=xxx`
- 退出登录使用受害者的账户登录 -> 浏览器被引导到`/login2`(2FA页面)
- 直接修改URL为`/my-account?id=`受害者ID -> 进入受害者账户页面
## 防御措施
- 密码验证后：Session 中仅标记 `awaiting_2fa = true`，不赋予任何资源访问权限，2FA 验证后：将 Session 升级为完整授权状态，才允许访问`/my-account` 等页面
- 对所有受保护页面，强制检查 2FA 是否已完成，未完成则重定向回验证页面

## 总结
- 2FA 形同虚设——密码验证通过后 Session 即已生效，直接修改 URL 跳过验证码页面即可访问受保护资源。
