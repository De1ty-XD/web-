# 实验名称：用户ID请求篡参数控制，密码泄露
## 基本信息
- **难度** 初级
- **类别** 访问控制
## 漏洞原理
- 用户管理面板ID参数值没有做身份验证
- 用户ID值被猜测出来
- 安身份证在前端密码被直接泄露
## 攻击步骤
- 登录自己的账号进入用户管理面板
- 发现ID参数值没有身份验证 -> 推举猜测出管理员账户为administrator
- 在URL修改ID参数进入administrator用户面板
- CTRL+U查看页面源码发现密码值直接暴露在前端 -> 复制密码直接在/login页面登录administrator
- 进入控制管理面板删除目标
## 防御措施
- 增加ID参数验证，应在服务端验证当前登录用户是否有权访问该 ID
- 使用不可被推举出来的ID参数，如GUID随机生成
- 管理控制用户不应该和普通用户使用同一个登录系统
- 密码参数身份验证等敏感行为全部在后台完成
## 总结
- 因为没有身份验证弱ID参数外加敏感数据直接暴露在前端，直接获得管理员帐户页面的访问权限，提供对特权功能的直接访问权限。
