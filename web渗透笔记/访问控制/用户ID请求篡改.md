# 实验名称：用户ID请求篡改
## 基本信息
- **难度** 初级
- **类别** 访问控制
## 漏洞原理
- 用户管理面板ID参数值没有做身份验证
- 用户ID值被直接暴露
## 攻击步骤
- 找到目标用户的博客 -> 点击链接Carlos查看URL上的ID参数（例：academy.net/blogs?userId=60ade642-89ae-4e9e-af3a-c9995a4c719c）
- 登陆自己的账户 -> 将URL上的ID参数修改为保存的受害者ID
- 获取目标API密钥
## 补充说明
- GUID（Globally Unique Identifier，全局唯一标识符）是一种用于标识信息的128位数字，GUID 只是增加了猜测难度，并不能替代服务端的权限校验。
## 防御措施
- 增加ID参数验证，应在服务端验证当前登录用户是否有权访问该 ID
- 避免在公开页面暴露敏感 ID
## 总结
- 通过获取目标ID在自己的用户后台修改ID值即可实现横向权限提升 ps:永远不要相信客户端传来的身份参数，身份验证必须在服务端完成。
